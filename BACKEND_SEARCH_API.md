# 검색 기능 백엔드 API 요구사항

## 개요
홈 화면의 전공자 목록에 검색 기능이 추가되었습니다. 백엔드에서 다음 파라미터를 처리해주세요.

## API 엔드포인트
`GET /major-profiles`

## 요청 파라미터

### 기존 파라미터
- `page`: 페이지 번호 (0부터 시작)
- `size`: 페이지 크기

### 새로 추가된 파라미터
- `searchType`: 검색 타입
  - `all`: 닉네임/학교/학과 통합 검색
  - `tag`: 태그 검색
- `keyword`: 검색어

## 요청 예시

### 1. 통합 검색 (닉네임, 학교, 학과)
```
GET /major-profiles?page=0&size=8&searchType=all&keyword=서울대
```
- 닉네임에 "서울대" 포함 OR
- 학교에 "서울대" 포함 OR
- 학과에 "서울대" 포함

### 2. 태그 검색
```
GET /major-profiles?page=0&size=8&searchType=tag&keyword=AI
```
- 태그에 "AI"가 포함된 전공자만 검색
- 프론트엔드에서 `#`은 제거하고 전달합니다

### 3. 검색어 없음 (전체 조회)
```
GET /major-profiles?page=0&size=8
```
- 기존과 동일하게 전체 목록 반환

## 구현 요구사항

1. **searchType과 keyword가 모두 있을 때만 검색 적용**
   - 둘 중 하나라도 없으면 전체 조회로 처리

2. **검색 시 대소문자 구분 없이 처리** (권장)
   - 예: "AI", "ai", "Ai" 모두 동일하게 검색

3. **부분 일치 검색**
   - 예: "서울" 검색 시 "서울대학교", "서울시립대" 모두 검색

4. **응답 형식은 기존과 동일**
   ```json
   {
     "success": true,
     "data": {
       "content": [...],
       "totalPages": 5,
       "totalElements": 42,
       "size": 8,
       "number": 0
     }
   }
   ```

## 프론트엔드 동작

- 사용자가 검색어 입력 후 Enter 또는 검색 버튼 클릭
- `#`으로 시작하는 검색어 → 태그 검색
- 그 외 검색어 → 통합 검색 (닉네임/학교/학과)
- 태그 버튼 클릭 → 자동으로 `#태그명`으로 검색

## 추가 고려사항

1. **검색 성능 최적화**
   - 데이터베이스 인덱스 추가 권장 (nickname, university, major, tags)

2. **검색어 길이 제한** (선택사항)
   - 너무 짧은 검색어(1자)는 성능 이슈가 있을 수 있음
   - 필요 시 최소 2자 이상 검색어 요구

3. **특수문자 처리**
   - 검색어에 특수문자가 포함될 수 있으므로 SQL Injection 방지 필요
